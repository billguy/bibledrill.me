wowBook Documentation

Overview

wowBook is a jQuery plugin that let's you publish a book on your website with awesome page flipping effects.

How to use

Create a wowBook in your site requires 4 steps.

1 - Find the directory 'wow_book' that comes in the zip file you downloaded from CodeCanyon. Upload this directory to your webserver document root.

2 - Include JQuery, the wowBook script and the wowBook CSS on your page:

3 - Create the book content using html. Create a div to hold the book pages. Inside that div, create one div for each page of the book.

4 - Call the plugin to turn your html into a awesome wowBook!

In this example we created a the book with 500 pixels height and a 600 pixel width. Each page will have a 300 pixels width.

You can pass as many different options as you'd like by following each one with a comma (except the last one).

Check your book. Go and see each page to check if everything is fine. Since you can put any html inside the page, it's possible that some combination could 'break' the book. The best way is to check it before put it online. Look below some points that you have to consider when design a book.

Design notes

Some things to take into consideration when designing / styling your book.

  • If the element passed to wowBook is not positioned, wowBook will make it position to 'relative'.
  • Every element that contains a page's content will be put inside another element, a page container.
  • An element that contains a page's content cannot have margins.
  • An element that contains a page's content will have the css "box-sizing" set to "border-box". Javascript is used to set the element's dimensions if the browser does not support it.
  • An element that contains a page's content will have a class "wowbook-page-content". His parent will have a class "wowbook-page".
  • If you need to change a style to all pages (like padding or background), change the class "wowbook-page-content".
  • Every element that contains a page's content will have the css "overflow" set to "hidden".

Flash and Video

To create the flip effect the plugin use css transformations. Unfortunately, browsers doesn't like when you mix css transformations with flash and video. In order to avoid any problems, put a CSS class 'wowbook-basic-page' in every page that have a flash object or a video tag. This will force the plugin to use only the basic(a.k.a boring) page flipping effect.


Gutter shadow

The gutter shadow is a div placed over the book page's content.

This can be a problem if the div is over a element that the user may interact using the mouse (like selecting a text of the page, a link or a input field). In those cases, you can:

Remove the gutter shadow

set gutterShadow configuration option to false

Put the page content above the gutter

Put the page content inside a positioned div with z-index above the gutter shadow div (a number like 10 will do the trick)

Make the gutter shadow smaller

You can use CSS to change the width of the gutter shadow

For instance, to change the width to 20 pixels, put the snippet below in you css file or inside a style tag:

				.wowbook-gutter-shadow {
					width: 20px;
				}

Responsive

Wowbook gives you a configuration option called "scaleToFit" to help you make your book to adapt to various screen sizes.

Pass a jQuery Object, a dom element or a jQuery selector to this option and the plugin will create a book that fits inside this dom element dimensions.

				$("#book").wowBook({
				  width: 1000,
				  height: 500,
				  scaleToFit: "#some_element"
				});

In the example above the book will be created and then scaled to fit inside the dimensions of "#some_element", keeping the original aspect ratio. So if "#some_element" have width of 800px and height of 500px, the book will be scaled to have 800px width and 400px height.

Now you can use CSS media queries to define the dimensions of "#some_element" for every screen/device, and the book will follow these dimensions.

If the dimensions of "#some_element" changes when the user resize the browser window (either manually or by changing the device orientation), the book scale will be updated accordingly.

  • - You still need to define the configuration options 'width' and 'height'.
  • - The DOM element doesn't need to contains the book element.
  • - Only the book dimensions are affected, not the book contents or its position.

Element's padding is not considered when the plugin calculates the height/width. You can use this to create space for a menu, a toolbar or a footer. For example, in the css below, the body element will have a 50px padding. If we pass the body to the plugin in the "scaleToFit" option, the book will be scaled to fit inside the browser window, but will have a 50px space around it.

				/* make the body fill the whole screen */
				html, body { height: 100%; margin 0; }

				body {
					/* with border-box, padding is included in the  body's height/width */
					-webkit-box-sizing: border-box; /* Safari/Chrome, other WebKit */
					-moz-box-sizing: border-box;    /* Firefox, other Gecko */
					box-sizing: border-box;

					padding: 50px;
				}
If you need to change the book dimensions programmatically, use the method setDimensions.

PDF support

Wowbook supports pdf files using the library PDF.js from Mozilla. PDF.js requires 3 files to be included but for your convenience 1 file containing all these files is provided.

The first step to use woobook with a pdf file is to include the PDFjs library. Remember to include the file before include wowbook

Now set the configuration option "pdf" to the url of the pdf file.

				$("#book").wowBook({
				  // ...
				  // other configuration options
				  // ...
				  pdf: "some_pdf_file.pdf"
				});

Wowbook will create the book using the page dimensions from the PDF file. If you also set the configuration option scaleToFit, the book will be resized accordingly.

Limitations

  • - PDF.js works in the following browsers: Firefox, Chrome, Opera, IE10+. It has limited support for Safari, Android and IE9. Not supported on IE8. For more information click here.
  • - By default, PDF.js cannot load a pdf from another server(ie from another domain) because of the Same origin policy. For more information click here.
  • - wowbook supports only html links in the pdf (for now).
  • - It can take several seconds to render a pdf page. It's normal, if you want to avoid this convert your pdf into images.
  • - Each pdf page that is rendered can use a good amount of memory. By default wowbook will only keep 6 pages in memory to avoid consume too much memory. You can change this behavior by setting the configuration option "pagesInMemory".
  • - Click here to see what you can do to optimize your pdf file.

Fullscreen

Wowbook can help you to make your site/book full screen if the browser supports the Full Screen API.

First, you need to choose what element on the page will be used to occupy the whole screen. By default wowbook will use document element, which means your whole website will take the screen. If you want to change this behavior, set the configuration option "fullscreenElement"

				$('#mybook').wowBook({
				  height : 500, width  : 600,
				  // the element "#book_container" will be show in fullscreen mode
				  fullscreenElement: "#book_container"
				});

Fullscreen control option

The configuration option "controls" let's you turn a html element into a button that will enable/disable full screen mode, like this:

				$('#mybook').wowBook({
				  height : 500,
				  width  : 600,
				  controls : {
				    // the element "fullscreen_button" now will
				    // enable/disable full screen mode when clicked
				    fullscreen : "#fullscreen_button"
				  }
				});

Fullscreen using the plugin API

The second way is to use the plugin api, calling the methods enterFullscreen, exitFullscreen and toggleFullscreen, like the example below: 

				$('#mybook').wowBook({
				  height : 500,
				  width  : 600
				});
				// get the book object
				var book = $.wowBook("#mybook");
				book.enterFullscreen(); // request the user permission to enter in fullscreen mode
				book.exitFullscreen(); // leave fullscreen mode
				book.toggleFullscreen(); // alternate between enter/exit fullscreen mode

Styling for fullscreen

When the browser is in fullscreen mode, wowbook will add the CSS class "fullscreen" to the following elements:

  • - Book root element, ie, the element you passed to the plugin to create the book
  • - The element passed in the configuration option "fullscreenElement"

Additionally, the CSS class "wowbook-disabled" will be added to the element used in the configuration option "controls.fullscreen" when in fullscreen mode.

These changes can be used to change the style of your site. For instance, in the live preview this is used to change the image on the button "fullscreen" in the toolbar.

Right to left

To enable right to left support, just set the configuration option "rtl" to true.

				$('#mybook').wowBook({
				  height : 500, width  : 600,
				  // ... other options ...
				  rtl: true
				});

Using the wowbook API

You can use the api to have greater control over the book capabilites at runtime.

To use the api, you need to use the "$.wowBook" function to get the wowBook object related to the book you created.

				// create the book
				$("#some_element").wowBook({
					width: 500, height: 500
				});

				// get the book object reference
				var book = $.wowBook("#some_element");

				// now you can call methods and properties
				book.startSlideShow();
				book.stopSlideShow();
				alert("This book has "+book.pages.length+" pages.");

Alternatively, you can use a jqueryUI style. To call methods:

				// create the book
				$("#some_element").wowBook({
					width: 500, height: 500
				});
				// get the book object reference
				$("#some_element").wowBook("zoom", 1);

To set and get book properties:

				// get the book object reference
				$("#some_element").wowBook("prop", "pageHeight"); // return 500
				$("#some_element").wowBook("prop", "shadows", false); // disable shadows
				$("#some_element").wowBook("options", "shadows", false); // 'options' works too

Glossary

page index
All the pages in the book have a associate page index. The cover has index 0, the second page has index 1, and so on.
spread
two adjacent, facing pages in a newspaper, magazine, or book. In wowBook, pages on the same spread are pages that are exhibited together. The book's covers are exhibited alone, but the other pages are always viewed in pairs. Pages in the same spread are pages 1 and 2, 3 and 4, 5 and 6, and so on. The front cover (page 0) and back cover (if exist) has no other page in the spread.

Pages

Pages:

All the pages in the book have a associate page index. The cover has index 0, the second page has index 1, and so on. Some configuration options and methods accepts a negative page index. A negative page index is assumed to be relative to the end of the book — that is, an index of -1 indicates the last page, -2 is the second to last page, and so on.

Double pages

You may need a page that expands across two pages, what we call it a "double page". Since you can't have a page that spans the whole book, you have to split the page in two pages. The good news is that wowbook can split the page for you.

You have two ways to tell wowbook when a page is a double page and should be splitted. You can add a attribute "data-double" or you can add a CSS class "double".

In the example above, the pages "page2" and "page3" will be splitted in two, leaving the book with a total of 7 pages. This is what happens when the wowbook "splits" the page "page2":

  • - First the div "page2" is cloned.
  • - Then both divs are set with a width equivalent to the book's width. But only half of each element will be visible.
  • - Each div is wrapped in a container div and both containers are added to the book, side by side.

It's important to keep in mind that a clone of the element will be created, and only half of each is visible.

For instance, suppose you have a double page with a input element in the middle of the page, ie, half of the input element is on the book's left side the half is on the right side. After the cloning, the page will have 2 input elements, one for left side and other for the right side. Visually, both elements together will look like one element.

The user starts to type inside the input element. Once the text he is typing pass the input element's half, he will no longer be able to see what he is typing. The input element will look "broken" to him. The solution is to not put a input element in the middle of the book.

You can change the CSS class used to define a double page. Set the configuration option "doublePages" to a jquery selector. The plugin will consider a double page every page that matches the selector. For instance, consider the javascript below applied to the html in the previous example: 

				// create the book
				$("#book").wowBook({
					width: 500, height: 500,
					doublePages : "#page5, .foobar"
				});

With the configuration above the double pages will be "page2", "page4" and "page5". The "page2" div because it has the attribute "data-double". Divs "page4" and "page5" will be considered a double page because both match the selector "#page5, .foobar".

Lazy loading

Lazy loading allows you to defer the loading of a page contents until the page is shown. This is especially useful in books that contains a lot of images.

You define a lazy loaded page you use attributes "data-src" and "data-image". Look the example below.

The page "page2" has the "data-image" attribute. That means that when the page "page2" is shown the image "foo.jpg" will be loaded and the proper img tag will be inserted in the page.

Pages "page3", "page4" and "page5" have a place holder "{{x}}" that will be replaced for the page index(index, not the page id). That means that "page3" will load the url "./foo_3.html". Because the place holder for "page4" and "page5" have 3 "x"s instead of one, the replacement will be padded with 3 zeroes. The page "page4" will load the the image in the url "./foo?image_id=004" and "page5" will load the image "./foo?image_id=005", despite having the same "data-image" attribute.

Memory usage

Lazy loading allows you to defer the loading of a page contents, but once the page is loaded, it stays in the memory until the book is destroyed (or the user leave your website). To minimize memory usage, you can ask wowbook to keep only a maximum number of pages in memory.

You should use this feature if your book has a lot of pages, mostly images. To enable this, use the configuration option "pagesInMemory".

If you want to prevent a page from being unloaded (for instance, your page may have a form) you can add a attribute "data-keep='true'" to the page's div or set the configuration option "pagesToKeep".

In the example above the first 2 pages will not be unloaded, because one have the data-keep attribute set to true and the other matches the selector in the option "pagesToKeep".

Page flipping effects

wowBook gives you 3 types of page flipping effects:

  • hardPage
  • softPage
  • basic - is used as fallback when the browser does not support the other 2 types.

By default, all pages are 'softpages'. There are several ways to make a page a 'hardPage':

  • set the configuration option hardPages to 'true'. This will make all pages to be 'hardPages'.
  • set the configuration option hardPages to an array with the index of the pages that will be 'hardPages'.
  • give a page element the class 'wowbook-hardpage'
For convenience, you can set the option 'hardcovers' to 'true' to make the book covers 'hardPages'.

Thumbnails

Wowbook can display thumbnails of your book pages. When the user click on a page thumbnail, the book will show that page. To tell the plugin to create thumbnails set the configuration option thumbnails to true. This will create the thumbnails but will not display them. To display the thumbnails you have two options. The first one is to use the configuration options controls to turn a regular html element into a control that will show/hide the thumbnails, like this : 

				$('#mybook').wowBook({
				  height : 500,
				  width  : 600,
				  thumbnails : true, // we can omit that because 'true' is the default value to this option
				  controls : {
				    // the element "thumb_button" now show/hide the thumbnails when is clicked
				    thumbs : "#thumb_button"
				  }
				});

The second way is to use the plugin api, calling the methods showThumbnails, hideThumbnails and toggleThumbnails, like the example below: 

				$('#mybook').wowBook({
				  height : 500,
				  width  : 600,
				  thumbnails : true, // we can omit that because 'true' is the default value to this option
				});
				// get the book object
				var book = $.wowBook("#mybook");
				book.showThumbnails();
				book.hideThumbnails();
				book.toggleThumbnails(); // alternate between show / hide the thumbnails

Automatically generated thumbnails or Image thumbnails?

You can create the images used as thumbnails or let the plugin generate the thumbnails automatically for you.

Compared to image thumbnails, thumbnails generated by the plugin has 3 possible drawbacks:

  • they use more browser resources (each thumbnail is a copy of the respective page)
  • are slower to render (the browser has to render the page and scale down, as opposed to just draw a image)
  • because browsers are not always good in the "scale down" part, the end result can be very ugly

Which method you should use? During development, let the plugin handle the thumbnails. After you site is finished, If you're not satisfied with the plugin generated thumbnails you can give your own thumbnails as images.

Automatic thumbnails are default. If you want to create the thumbnails manually, create a CSS sprite with your thumbnails and use the configuration option thumbnailsSprite to use it.

Thumbnails size

You have 3 ways to set the thumbnails size.

You can set using the configuration option thumbnailScale. This option is a number between 0 and 1 that specifies the proportion between the page size and the thumbnail size. For instance, if your book has a page with 400px width and 500px height, the default value of 0.2 will generate a thumbnail with 80px width and 100px height.

You can set the thumbnail size using CSS. Just set the width or height of the CSS class ".wowbook-thumbnail" and you're done. If you add this snippet to your CSS 

			.wowbook-thumbnail {
			  width : 100px; /* you can also use "ems" */
			}
You will have thumbnails with 100px width. The height will be automatically set to keep the aspect ratio, so you only have to define the width or the height.

This is useful if you're using media queries to create a responsive site and want different thumbnails sizes for different resolutions.

The third way to set the thumbnail size is using the configuration options thumbnailWidth to set the thumbnail width or "thumbnailHeight" to set the thumbnail height. Like when you use CSS, you only need to set the width OR the heigth, and the plugin will figure out the other value to keep the aspect ratio. For instance, this javascript code

				$(document).ready(function(){
				  $('#mybook').wowBook({
				    height : 500,
				    width  : 600,
				    thumbnailHeight : 100
				  });
				});

will create thumbnails with 100px height and 60px width. The width will be automatically set to keep the aspect ratio.

If you set the size using "thumbnailWidth" and "thumbnailHeight", the plugin will ignore if you set through CSS or "thumbnailScale". If you set the size using CSS the plugin will ignore "thumbnailScale".

Positioning your thumbnails

The thumbnails are created inside a container element, by default this container is appended to the "body" element. You can change that by setting the configuration option "thumbnailsParent".

You can choose where the thumbnails will be placed inside the "thumbnailsParent" element using the option "thumbnailsPosition". This option can have the values "left", "right", "top" and "bottom".

You can set the dimensions of the thumbnails container using the configuration options thumbnailsContainerWidth and thumbnailsContainerHeight. Changing the dimensions of the container element and the parent's element should give you the freedom to make the thumbnails adapt to your design without problems.

Customizing with CSS

Below is listed how to use CSS to customize your thumbnails.

				/* Change thumbnails container styles */
				.wowbook-thumbnails {
					background: none repeat scroll 0 0 #FFFFFF;
					box-shadow: 0 0 15px black;
				}

				/* Change button size to 50px */
				/* When thumbnailsPosition is "left" or "right" */
				.wowbook-thumbnails .wowbook-button {
					width: 100%; /* default */
					height: 50px;
				}
				.wowbook-vertical.wowbook-thumbnails .wowbook-clipper {
					top: 50px; bottom: 50px; /* same as the buttons height */
				}
				/* When thumbnailsPosition is "top" or "bottom" */
				.wowbook-horizontal.wowbook-thumbnails .wowbook-button {
					width: 50px;
					height: 100%; /* default */
				}
				.wowbook-horizontal.wowbook-thumbnails .wowbook-clipper {
					margin: 0 50px; /* same as the buttons width */
				}


				/* Change a style common to both buttons */
				.wowbook-thumbnails .wowbook-button {
				  background-color: green;
				}
				/* Change the image (or anything else) of the back button */
				.wowbook-thumbnails .wowbook-button.wowbook-back {
				  background-image: url(back_button_image.png);

Something is wrong with my thumbnails!

Here are some problems you might find while working with thumbnails

They are ugly:Browsers can generate very ugly thubmnails when scaling down the elements. Try change the thumbnails dimensions.If nothing solves, your last resort it's creating sprites.

Background images being displaced on IE8: If you try to customize the thumbnails CSS, make sure to check on IE8. One bug i found is that it misplace the background image when the element has styles zoom different from 1 and is absolute positioned.

Thumbnails are different from the book!If you are using plugin generated thumbnails and they're are different from the book, check your CSS. A thumbnail is a copy of the page, but is inside another element. Some CSS rules applied to elements inside the book may not being applied on the thumbnail. For instance, suppose you have a CSS rule like this: 

				#mybook h1 {
				  color: red; font-size: 100px;
				}
A book page with a h1 element will have a big red font. When this page is copied to the thumbnail, the h1 element will no longer be inside the "mybook" element, and it "loses" the big red font. You have to write your CSS taking this into consideration. One (not necessarily good) solution would be 

				#mybook h1, .wowbook-thumbnail h1 {
				  color: red; font-size: 100px;
				}

Page flip sound

wowBook uses the HTML5 audio element to play a "flipping page" sound when the user flips a page. If the user browser does not support it, nothing happens.

Changing the sound file

If you want to change the sound file it's recommended that you provide 2 versions of the file, one in mp3 format and the other in Ogg Vorbis format. Then pass those files names in the config option "flipSoundFile".

Put those files on you server and use the flipSoundFile configuration option, like this:

				$('#mybook').wowBook({
				  // ... other configuration options
				  ,flipSound     : true
				   // array with the files
				  ,flipSoundFile : ["my_sound_file.mp3", "my_sound_file.ogg" ]
				  // where are these files. This value will be prepended in each file name.
				  // Don't forget the ending "/"
				  ,flipSoundPath : "www.mysite.com/sounds/"

				  // ... other configuration options
				});

the browser will search for the appropriate audio file format that he supports.

Using another audio player

To use another audio player, you can use the onPlayFlipSound callback to tell the player when to play the sound, like this: 

				$('#mybook').wowBook({
				  // ... other configuration options
				  , onPlayFlipSound: function(book){
				    // ...
				    // code that makes your player play the flip sound
				    // ...
				    return false; // if this callback returns false, wowBook will not play the flip sound
				  }
				});

Sections

You can divide a wowBook into sections by defining elements inside the book that mark a section's start. When the wowBook is showing a page that contains a 'start section element', the browser url hash will be changed to "#book_element_id/section_element_id" instead of "#book_element_id/page_index".

A practical example: suppose you created a magazine, and this magazine has an article that begins on page 3 and another one that begins on page 7. In addition, you created links to each article using the id of the h1 elements, something like this:

Normally, when the user moves from page 2 to page 3 (the first page of "article 1"), the url hash in the browser address bar is set to "#myzine/5." But if you define the element "#article1" as a section start mark, the url hash will be set to "#myzine/article1."

The advantage in using sections is that now the user can bookmark a link to article instead of bookmark a link to the page 3 of the magazine. If you want to reorder the magazine's content and move the article to the page 10, the bookmark to the article keeps valid in this case.

Advancing the pages of the book sets the the url hash to "# magazine/awesome_article/2", "# magazine/awesome_article/3," and it goes this way until the next session start. The table below shows the page index and the relative url hash.

".wowbook-section"
string or array of strings
the section's definition, as in the configuration option "sections". If you want to change this, call the method findSections
findSections
s : a section's definition like the configuration option "sections".

Search in the book for the sections info accordingly with the definitions passed in parameter S and stores this info at the "sections" property.

If the parameter S is provided, wowBook will set the property sectionDefinition to this value and use it trough the method. Otherwise the property sectionDefinition is used as default.

Call this method when you made something that changed the sections in the book. This method already is called when a book's page is inserted or removed, use it only if you know what you are doing.
